# Документ требований

## Введение

Сервис для удаления личной информации из телеграм ботов - это веб-платформа, которая позволяет пользователям проверить наличие своих персональных данных в различных телеграм ботах и получить инструкции по их удалению. Система интегрируется с 5 различными API ботов для поиска информации по номеру телефона, email или документам (ИНН, СНИЛС, паспорт).

## Требования

### Требование 1

**Пользовательская история:** Как пользователь, я хочу проверить наличие моих персональных данных в телеграм ботах, чтобы узнать где хранится моя информация.

#### Критерии приемки

1. КОГДА пользователь заходит на главную страницу сайта, ТО система ДОЛЖНА отобразить форму для ввода персональных данных
2. КОГДА пользователь вводит номер телефона, email, ИНН, СНИЛС или паспортные данные, ТО система ДОЛЖНА валидировать введенные данные
3. КОГДА пользователь нажимает кнопку "Проверить", ТО система ДОЛЖНА инициировать поиск по всем 5 API ботов
4. ЕСЛИ данные не прошли валидацию, ТО система ДОЛЖНА показать сообщение об ошибке

### Требование 2

**Пользовательская история:** Как пользователь, я хочу видеть результаты поиска моих данных, чтобы понимать в каких ботах есть моя информация.

#### Критерии приемки

1. КОГДА система завершает поиск по всем API, ТО система ДОЛЖНА отобразить результаты в структурированном виде
2. КОГДА отображаются результаты, ТО названия ботов ДОЛЖНЫ быть зашифрованы (например, "Бот A", "Бот B" и т.д.)
3. КОГДА найдена информация в боте, ТО система ДОЛЖНА показать какие именно данные найдены
4. ЕСЛИ информация не найдена ни в одном боте, ТО система ДОЛЖНА показать соответствующее сообщение

### Требование 3

**Пользовательская история:** Как пользователь, я хочу получить инструкции по удалению моих данных, чтобы самостоятельно удалить информацию из ботов.

#### Критерии приемки

1. КОГДА пользователь нажимает кнопку "Удалить" рядом с найденными данными, ТО система ДОЛЖНА перенаправить на страницу с инструкциями
2. КОГДА отображается страница инструкций, ТО система ДОЛЖНА показать пошаговое руководство для каждого бота
3. КОГДА отображаются инструкции, ТО система ДОЛЖНА использовать зашифрованные названия ботов
4. КОГДА пользователь просматривает инструкции, ТО система ДОЛЖНА предоставить четкие шаги: активация бота, переход в "мой профиль", выбор "удалить информацию обо мне"

### Требование 4

**Пользовательская история:** Как администратор системы, я хочу интегрировать 5 различных API ботов, чтобы система могла выполнять поиск по всем доступным источникам.

#### Критерии приемки

1. КОГДА система инициализируется, ТО система ДОЛЖНА подключиться ко всем 5 API ботов
2. КОГДА выполняется поиск, ТО система ДОЛЖНА отправить запросы параллельно ко всем API
3. ЕСЛИ один из API недоступен, ТО система ДОЛЖНА продолжить работу с остальными API и уведомить о недоступности
4. КОГДА API возвращает данные, ТО система ДОЛЖНА обработать и структурировать ответы

### Требование 5

**Пользовательская история:** Как пользователь, я хочу чтобы система была безопасной и не сохраняла мои персональные данные, чтобы защитить мою приватность.

#### Критерии приемки

1. КОГДА пользователь вводит персональные данные, ТО система НЕ ДОЛЖНА сохранять эти данные в базе данных
2. КОГДА выполняется поиск, ТО система ДОЛЖНА использовать данные только в оперативной памяти
3. КОГДА сессия пользователя завершается, ТО система ДОЛЖНА полностью очистить все персональные данные из памяти
4. КОГДА система логирует операции, ТО логи НЕ ДОЛЖНЫ содержать персональные данные пользователей

### Требование 6

**Пользовательская история:** Как владелец бизнеса, я хочу подготовить систему для будущей интеграции платежей, чтобы монетизировать сервис.

#### Критерии приемки

1. КОГДА пользователь заходит на сайт, ТО система ДОЛЖНА показать информацию о тарифах (пока без возможности оплаты)
2. КОГДА отображаются тарифы, ТО система ДОЛЖНА показать заглушку "Оплата будет доступна позже"
3. КОГДА архитектура системы проектируется, ТО система ДОЛЖНА предусматривать место для интеграции платежной системы
4. КОГДА пользователь пытается воспользоваться сервисом, ТО система ДОЛЖНА пока предоставлять бесплатный доступ
